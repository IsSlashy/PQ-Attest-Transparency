PQ-Attest-Transparency — client-side receipt verifier (WASM)

This page verifies a confidential-inference session receipt entirely in your browser: post-quantum STH signature (SLH-DSA), Merkle inclusion, session binding, and that the build's root matches an independently-trusted one. No server is contacted to decide the verdict; the WASM verifier uses no randomness.

      The honest receipt verifies ✅ — the browser checks the quote signature, the STH
      signature, Merkle inclusion, AND that a threshold of a pinned witness set cosigned the
      STH. Flipping one signature byte ❌ (signature mismatch); dropping cosignatures below the
      threshold ❌ (quorum not reached). The browser verifies the quorum mechanism
      honestly — but note: in THIS demo the witness keys are bundled into
      sample-receipt.json by the provider's emit tool for convenience.
      A real client MUST pin the witness set out of band, independent of the receipt source, or
      the anti-split-view property collapses.
    

  


  

    About the trust model
    

      The receipt proves non-equivocation (the build is in a public,
      append-only log) and HNDL-safe binding (X25519+ML-KEM-768). It does
      not prove the build is honest — only that it cannot be served to you in
      secret. This page verifies the witness quorum itself — a threshold of the
      client-pinned witness keys must have cosigned the STH. The hardware-root and witness public
      keys are still pinned out of band; that distribution channel is the remaining trust
      assumption (see THREAT-MODEL.md).
